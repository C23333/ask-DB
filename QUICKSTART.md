# Quick Start Guide

Get DB Assistant running in 5 minutes!

## Prerequisites

- Go 1.21+ installed
- Node.js 18+ installed
- Oracle database access (host, port, credentials)
- LLM API key (OpenAI, Claude, etc.)

## Step 1: Configure Backend (2 minutes)

```bash
cd backend

# Create configuration
cp .env.example .env

# Edit .env with your details
nano .env
```

**Minimum required settings:**
```env
ORACLE_USER=your_readonly_user
ORACLE_PASSWORD=your_password
ORACLE_HOST=your_database_host
ORACLE_PORT=1521
ORACLE_SID=your_sid
# Optional: schema owner to inspect (defaults to ORACLE_USER)
ORACLE_SCHEMA=target_schema_owner
LLM_API_KEY=your_api_key_here
JWT_SECRET=random-secret-string
```

## Step 2: Start Backend API (1 minute)

```bash
# From backend directory
go run cmd/api/main.go
```

Expected output:
```
Starting DB Assistant API Server...
Database connected successfully
Server listening address=:8080
```

## Step 3: Setup & Run Frontend (2 minutes)

In a new terminal:

```bash
cd web

# Install dependencies (first time only)
npm install

# Start development server
npm run dev
```

## Step 5: Try It Out!

1. **Generate SQL from Natural Language:**
   - Type in the left panel: "Show me all users with their total purchases"
   - Click "Generate SQL"
   - AI will create the SQL query

2. **Execute the Query:**
   - Click the "Execute" button
   - Results appear in the "Results" tab
   - See execution time and row count

3. **Debug if Needed:**
   - If query fails, click "Debug"
   - AI suggests fixes
   - Click "Apply" to use the suggested SQL

4. **Save for Later:**
   - Click "Save Query"
   - Give it a name: "User Purchases"
   - Access from "History" tab anytime

## Common Tasks

### Create a Read-Only Oracle User

If you don't have a read-only user yet:

```sql
-- Connect as DBA
sqlplus / as sysdba

-- Create read-only user
CREATE USER db_assistant IDENTIFIED BY your_password;
GRANT CREATE SESSION TO db_assistant;
GRANT SELECT ON schema.* TO db_assistant;  -- Replace 'schema' with your schema name

-- Test connection
sqlplus db_assistant/your_password@your_sid
SELECT COUNT(*) FROM user_tables;
```

### Get OpenAI API Key

1. Visit https://platform.openai.com
2. Sign up or login
3. Go to API keys section
4. Create new API key
5. Copy and paste into `.env` as `LLM_API_KEY`

### Test Database Connection

```bash
# From backend directory
go run -h  # Just to verify Go works

# Test Oracle connection
sqlplus your_user/your_password@your_sid
SELECT 'Connection successful!' FROM dual;
```

### Stop the Servers

- **Backend:** Press `Ctrl+C` in the backend terminal
- **Frontend:** Press `Ctrl+C` in the frontend terminal

## Troubleshooting

### "Cannot connect to Oracle"
- Verify credentials in `.env`
- Check if Oracle is accessible: `sqlplus user/password@sid`
- Verify host and port are correct

### "LLM API error"
- Verify API key in `.env` is correct
- Check API key has available quota
- Try with a simpler prompt first

### "Port already in use"
- Backend uses 8080, frontend uses 5173
- If already in use:
  ```bash
  # Kill process using port 8080
  lsof -i :8080
  kill -9 <PID>
  ```

### "Module not found" errors
- Backend: Run `go mod download` in backend directory
- Frontend: Run `npm install` in web directory
